All components follow a mobile first, content driven approach. Everything is intended to be touch friendly with large hit areas (where feasible).

Browser support

Our web application is built using SCSS to create modular components with BEM syntax. These components are generic enough to be reusable. This does result in support the following browsers

  • Internet Explorer 10+
  • Firefox (latest)
  • Chrome (latest)
  • Safari 7+
  • Android 2.2+
  • iOS 7+
  • Windows Phone 8+

While not a responsive or mobile-focused project, Mobile Safari and Chrome for Android should render just fine. Support for Linux-based browsers is not strictly guaranteed, but accounted for whenever possible.

However the application pages are built with progressive enhancement in mind, so older browsers (such as Internet Explorer 7. 8 and feature phones) should still have access to the content. It just may not look as good as using the latest browser, and there will be some layout issues.

Dependencies

  • Our styles are built with SCSS.
  • We use Font-awesome, Octicons and Blacktie icon fonts when ever neccessary.
  • Autoprefixer is used to generate vendor-prefixed declarations and is handled via our Gulpfile.
